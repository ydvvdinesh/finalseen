ATS-friendly resume checklist:
Visual & Layout
•	Margins: 1" (never < 0.5"); keep white space.
•	Length: 1 page (student/early career); CVs can be longer.
•	Font: Black, easy to scan (Arial, Calibri, Gill Sans MT, Georgia, Times New Roman, Verdana).
•	Sizes: Body 10–12 pt; Name 14–18 pt.
•	Emphasis: Consistent bold for headers/titles; avoid italics/underlines.
•	ATS: No tables/columns/text boxes/graphics/headers/footers/emojis.
•	Bullets: Action-oriented, not paragraphs.
Contact (Header)
•	Name (bold)
•	Email (professional)
•	LinkedIn URL (custom) + Portfolio/Website (if any)
•	Phone (answer professionally)
•	Address: omit; add city only if it helps (local/federal roles).
Sections (Suggested Order)
•	Summary (optional, 2–3 lines): use if you have substantial experience; otherwise skip.
•	Education: Degree, Major, School, City, Grad month/year; Thesis (if grad). GPA if ≥3.0 (or list Major GPA if stronger).
•	Experience / Internships / Clinicals: Reverse-chronological; 3–5 bullets per role with metrics.
•	Projects (students): 2–3 relevant projects with tools/results.
•	Leadership & Activities / Organizations
•	Skills: Group by category (Programming, Tools, Methods).
•	Languages: e.g., Hindi, English (proficiency level).
•	Awards & Honors
Bullet Formula (use on every role/project)
•	Action verb + what + tools + metric + outcome
e.g., “Built churn model (Python, XGBoost) cutting attrition 14% in 3 months.”
Quality & Consistency
•	Tense: Past for past roles; present for current.
•	Consistency: Dates, dashes, spacing, capitalization.
•	Proofread: Spelling, grammar, punctuation.
•	File name: First_Last_Role_Resume.pdf; export to PDF unless told otherwise.

Professional email address — Use a mature handle (e.g., first.last), not casual nicknames.
Clear subject line — Make it short, specific, and searchable (avoid “Hello”).
Manners — Use polite language (“please,” “thank you”) to show respect.
Proper salutation — Address by name with correct title (Mr./Ms./Dr.) unless invited otherwise.
Tone — Keep it friendly, neutral, and calm; never write in anger or ALL CAPS.
Be concise — State the ask upfront, keep it brief, and answer all questions when replying.
No emoticons/abbreviations/fancy formatting — Avoid 😊, LOL/TTYL, colors, or unusual fonts.
Spelling/grammar/punctuation — Proofread carefully; don’t rely only on spell-check.
Full name & signature — End with name + role, phone, and optional links (LinkedIn/website).
Reply promptly — Respond within 24 hours or acknowledge and give an ETA.
Avoid unprofessional content — Never send jokes, chain letters, or anything offensive.
Handle confidentiality — Don’t email sensitive data; use secure channels when needed.

Machine Learning Key Concepts for Data Science Interviews

Types of Learning
1.	Supervised Learning (labeled data → predict output)
o	Regression → predict continuous values
	Algorithms: Linear Regression, Ridge/Lasso, Decision Trees (regression), Random Forest, Gradient Boosting (XGBoost, LightGBM, CatBoost), SVR, Neural Networks
o	Classification → predict categories
	Algorithms: Logistic Regression, KNN, Decision Trees, Random Forest, SVM, Naïve Bayes, Gradient Boosting, Neural Networks
2.	Unsupervised Learning (unlabeled data → find patterns)
o	Clustering → K-Means, Hierarchical, DBSCAN, Gaussian Mixture Models
o	Dimensionality Reduction → PCA, t-SNE, UMAP, Autoencoders
o	Association → Apriori, FP-Growth
3.	Reinforcement Learning
o	Agent learns via rewards/penalties
o	Algorithms: Q-Learning, Deep Q-Networks, Policy Gradient methods

Model Performance & Generalization
•	Bias-Variance Tradeoff →
o	High Bias = underfitting (too simple)
o	High Variance = overfitting (too complex)
•	Overfitting → model memorizes training data
•	Underfitting → model too simple, misses patterns
•	Cross-Validation → k-fold, stratified → prevents overfitting

Data Handling & Feature Engineering
•	Preprocessing
o	Missing values → mean/median imputation, KNN imputer
o	Outliers → removal, transformation
o	Normalization/Standardization → MinMaxScaler, Z-score
o	Encoding Categorical Data → One-hot, Label Encoding, Target Encoding
•	Feature Engineering
o	Feature transformations (log, polynomial)
o	Feature selection (correlation, Chi-square, Lasso)
o	Interaction features

Model Selection & Evaluation
•	Choosing Algorithms
o	Small, linear data → Logistic/Linear Regression
o	Complex, non-linear → Random Forest, Gradient Boosting
o	High dimensional (text, images) → SVM, Neural Networks
•	Evaluation Metrics
o	Classification
	Accuracy → good for balanced data
	Precision → minimize false positives
	Recall → minimize false negatives
	F1 Score → balance precision & recall
	ROC-AUC → ranking performance
	Confusion Matrix → TP, TN, FP, FN
o	Regression
	RMSE (Root Mean Squared Error)
	MAE (Mean Absolute Error)
	R² (Coefficient of Determination)
Hyperparameter Tuning
•	Techniques
o	Grid Search
o	Random Search
o	Bayesian Optimization
o	Hyperband / Optuna
•	Examples
o	Decision Trees → max_depth, min_samples_split
o	Random Forest → n_estimators, max_features
o	XGBoost → learning_rate, max_depth, subsample
Machine Learning Lifecycle (End-to-End)
1.	Data Collection – gather raw data
2.	Preprocessing – clean, handle missing values, normalize
3.	Feature Engineering – create & select features
4.	Model Selection – choose algorithm(s)
5.	Training & Validation – fit, cross-validate
6.	Evaluation – metrics, error analysis
7.	Hyperparameter Tuning – optimize performance
8.	Deployment – production integration
9.	Monitoring – track drift, retrain if needed

Advanced Interview Concepts
•	Regularization → L1 (Lasso), L2 (Ridge)
•	Ensemble Methods → Bagging (Random Forest), Boosting (XGBoost, LightGBM), Stacking
•	Imbalanced Data Handling → SMOTE, class weights, undersampling/oversampling
•	A/B Testing → compare models/strategies
•	Feature Importance → SHAP, LIME for interpretability
•	Overfitting Control → dropout, pruning, early stopping
Tip for Interviews:
When discussing a project:
•	Define the problem clearly
•	Justify algorithm choice
•	Explain metrics used and why
•	Mention how you handled overfitting/underfitting
•	Highlight real-world challenges (missing data, imbalance, deployment issues)

Deep Learning
Fundamental Concepts
•	Artificial Neural Networks (ANNs) → Input, Hidden, Output layers; Neurons = weighted connections + activation
•	Deep vs. Shallow Networks → Shallow = 1 hidden layer; Deep = multiple hidden layers
•	Activation Functions → ReLU, Sigmoid, Tanh, Softmax
•	Loss Functions → MSE, Cross-Entropy, Hinge Loss
•	Tensors → Core data structure (multi-dimensional arrays greater than equal to 3, used in TensorFlow/PyTorch)
1 dim – Vector
2 dim – Matrix
3 dim - tensor

Model Architectures
•	Convolutional Neural Networks (CNNs) → Convolution, Pooling, ReLU, Fully Connected Layers → Image, vision tasks
•	Recurrent Neural Networks (RNNs) → Sequential data (text, time series); includes LSTMs, GRUs
•	Other Architectures (advanced) → Transformers (attention), Autoencoders (representation learning), GANs (generative modeling)

Training & Optimization
•	Forward Propagation → data flows through layers → output
•	Backpropagation → error gradients propagate back → weights updated
•	Gradient Descent → optimization algorithm
o	Variants: SGD, Mini-batch, Momentum, Adam, RMSProp
•	Learning Rate → step size in optimization (too high = divergence, too low = slow)
•	Regularization → Dropout, L1 (Lasso), L2 (Ridge), Early Stopping
•	Weight Initialization → Xavier, He Initialization

Common Challenges & Solutions
•	Overfitting → too good on training, poor on test → fix with dropout, regularization, more data
•	Underfitting → too simple model → fix with deeper network, better features
•	Data Requirements → need large datasets; use augmentation, transfer learning if limited

Advanced Concepts
•	Transfer Learning → use pre-trained networks (ResNet, BERT) for new tasks
•	Hyperparameter Tuning → learning rate, batch size, epochs, optimizer choice
•	Batch Normalization → stabilize training, faster convergence
•	Attention Mechanisms → focus on important input parts (NLP, vision transformers)

Applications
•	Natural Language Processing (NLP) → sentiment analysis, chatbots, translation, text summarization
•	Computer Vision → image classification, object detection, segmentation, facial recognition
•	Speech & Audio → speech-to-text, voice recognition
•	Recommendation Systems → embeddings, deep collaborative filtering

Interview Prep Tip:
Be ready to explain one or two project in depth:
•	Why you chose ANN/CNN/RNN
•	Which loss/activation you used
•	How you tuned hyperparameters
•	How you handled overfitting/underfitting
•	What metrics you used for evaluation

8 Step framework for Research:
• Problem Statement – Define in one clear sentence.
(e.g., “Can tweets with hashtags get more retweets than those without?”)
• Literature Review – Summarize 1–2 related papers/blogs.
• Hypothesis – State a testable claim.
• Data Collection & EDA – Find dataset, explore trends, clean data.
• Model Selection – Choose 1 baseline + 1 advanced approach.
• Validation – Train/test split, accuracy, F1, RMSE (depending on task).
• Documentation – Record process, graphs, results, code snippets.
• Sharing – Publish notebook/report + short post with insights.
